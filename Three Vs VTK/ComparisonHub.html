VTK.js vs Three.js

Jenna Horn

Hello, web browser! This is a project that I am doing to better understand the 3D rendering capabilities of both VTK.js and Three.js. My small demo projects are shown below.

Summary:

Three.js is a great program for 3D art graphics and video game style things. The way the library handles actors and lighting is very convenient. VTK is better for model viewing and algorithm rendering, as its performance with multiple actors is lacking as well as the ability for shadow mapping.

Three First Test

Make a cylinder in Three.js

I was able to construct a cylinder and a plane in the renderer as well as change the color of the cylinde, make it grow and shrink, and have it freely rotate.

VTK First Test

Make a cylinder in VTK.js

Since there is no direct Cylinder source, I rendered a line over a plane, made it grow and shrink, and also changed its color. I discovered the Tube Filter later and used it to make the cylinders in the later programs.

Three Performance Test

How many Cylinders can be rendered in the window?

This program allowed me to generate more cylinders into the scene to test what the renderer could do. I was also able to randomise the colors as they spawned to help differentiate them better.

VTK Performance Test

How many Cylinders can be rendered in the window?

This program can also generate more cylinders and test performance, but I opted out of randomising the colors because I would lose shadow rendering on them.

Three Light Test

How do the different light types render?

This allows the user to test the 4 different types of light rendering as well as see shadows cast onto other objects.

VTK Light Test

How do the different light types render?

This program tests the 3 different types of light rendering, but I was not able to cast shadows due to lack of vtk.js documentation.
